Item type JSON options (modder-focused guide)

This file explains what each currently-used option does in gameplay terms.
The examples are based on options actively used in `gamedata/baseconfig/types*.json` and `generateblocks*.json`.

How type inheritance works
- `parentType` makes a type inherit most values from another type.
- Child values override parent values.
- Some keys are local-only (not inherited): `isRotatable`, `categories`, `parentType`, `rotatablex+`, `rotatablex-`, `rotatablez+`, `rotatablez-`.

Common top-level options

"attachBehaviour"
- Adds behavior hooks to a block/item (job logic, custom interaction, special placement rules, etc).
- Can be a string (`"fuse"`) or object (`{ "id": "bomb", ... }`).
- Use this when a block should do more than just exist visually.

"attachmentProviders"
- Defines attachment-support volumes this block provides.
- Think of these as "mount points" other blocks can attach to.

"attachmentRequirements"
- Defines support that must exist for this type to be placeable.
- Good for wall-mounted / hanging / directional-support content.

"blockFertility"
- Marks a block as fertile soil type.
- Used by farming/crop systems when deciding what can grow on it.

"blueprints"
- Controls how this type behaves in blueprint planning/building.
- `type: "default"` = normal behavior.
- `type: "skip"` = blueprint system skips this type.
- `type: "parseAs"` + `value` = treat this type as another type in blueprint costs.
- `itemCountCost` can override item cost amount used by blueprint placement.

"canBuildUpon"
- Controls whether players can place blocks onto this block face.
- If omitted, default behavior is generally solid non-mesh blocks allow it.

"categories"
- Inventory/stockpile grouping tags.
- Affects sorting, filtering, and some UI/tool behavior.

"colliders"
- Defines actual collision/selection/placement shape.
- Use this to make thin/tall/custom-shape blocks behave correctly.

"color"
- Tint color multiplied into block/mesh visuals.
- Useful for color variants without new textures.

"colonyPointsMeal"
- Marks edible value for colony food systems.

"customData"
- Free-form mod data used by built-in and custom systems.
- Major currently-used keys are documented in the customData section below.

"destructionTime"
- How long (ms) block breaking takes.

"icon"
- Inventory/stockpile UI icon path.

"isDestructible"
- If false, the standard destroy interaction will not destroy it normally.
- Useful for protected/special-purpose blocks.

"isPlaceable"
- If false, item cannot be placed as a block.
- Use for resources/tools/intermediates that should stay inventory-only.

"isRotatable"
- Enables directional variants for a base type.
- Combine with `rotatablex+`, `rotatablex-`, `rotatablez+`, `rotatablez-`.

"isSolid"
- Core solidity flag.
- Affects collision/pathing/lighting/placement expectations.

"lightingTransparency"
- How much light passes through the block (0 = opaque, 255 = fully transparent).
- Key for windows, foliage, decorative meshes, etc.

"maxStackSize"
- Inventory stack cap for this item type.

"mesh"
- Custom mesh for rendering this type.
- Can be a string path, or object with `path` plus extra mesh data.

"meshFlipX"
- Mesh handedness flip toggle used by mesh import/render pipeline.

"meshRotationEuler"
- Mesh local rotation (`x`,`y`,`z` in degrees).

"movementCost"
- Adjusts pathing cost/speed feel over this type.
- Use to make surfaces feel slower/faster to path over.

"needsBase"
- Requires support below when placing (if no explicit attachment requirements are set).

"onHitParticleEffect"
- Controls hit/break particle style.
- Can be a single string type or weighted list of entries.

"onHitParticleEffectCount"
- Number of particles spawned on hit/break.

"onPlaceAudio"
- Sound played when placed.

"onRemove"
- Full drop table for block removal.
- Entries use `{ "type", "amount", "chance" }`.

"onRemoveAmount"
- Simple drop amount override (used with `onRemoveType` when no `onRemove` array exists).

"onRemoveAudio"
- Sound played when removed.

"onRemoveType"
- Simple drop type override (used when no `onRemove` array exists).

"parentType"
- Parent type for inheritance/fallback.

"pathingImpact"
- Pathing mode override:
  - `AsAir`
  - `AsSolid`
  - `AsUntouchableSolid`
  - `AsDoor`
- Use this when pathing behavior should differ from visual shape/solidity.

"requiredFertilityBelow"
- Required fertility flags on the block below for placement.
- Used by crop/farm-related blocks.

"rotatablex+" / "rotatablex-" / "rotatablez+" / "rotatablez-"
- Directional subtype names for rotatable blocks.

"sanctifiedPointsMeal"
- Sanctified meal points contribution for food systems.

"sideall"
- Base texture mapping for cube-style block rendering.
- `"SELF"` means use this type's own mapping name.

Nested keys currently used

"blueprints"
- "type": `default`, `skip`, `parseAs`
- "itemCountCost": explicit blueprint cost amount (`-1` = auto)
- "value": target type name when `parseAs`

"meshRotationEuler"
- "x", "y", "z"

"mesh" object
- "path": mesh file path
- "animation": optional mesh animation metadata
  - keys seen: `boundsMin`, `boundsMax`, `startAnimated`, `startLooping`, `startRandom`, `clip`, `events`
  - event keys seen: `time`, `type`, `layer`

"onRemove" entries
- "type", "amount", "chance"

"colliders" entries
- "min", "max"
- "impactsPhysics"
- "impactsPlayerSelection"
- "impactsBlockPlacement"

"attachmentProviders" entries
- "min", "max"

"attachmentRequirements" entries
- "axis", "min", "max", "offset"

customData keys currently used

"color"
- Used in torch/light config as HTML color.

"colors"
- Mesh color replacement mapping list (`"#RRGGBB->#RRGGBB"`).

"culledShadows"
- Enables shadow behavior optimized for culled mesh renderers.

"farShadows"
- Enables longer-distance shadow casting for relevant meshes.

"intensity"
- Torch/light intensity.

"merged"
- Torch/light merged config object (single merged light source setup).

"minerIsMineable"
- Marks type as minable by miner-related systems.

"minerMiningTime"
- Miner cooldown/time value for this type.

"offsetx", "offsety", "offsetz"
- Torch/light source offset.

"range"
- Torch/light radius.

"torches"
- Root torch/light config node.

"useHeightMap"
- Enables height-map shading features for mesh rendering.

"useNormalMap"
- Enables normal-map shading features for mesh rendering.

"volume"
- Torch/light audio volume.

Practical modding tips
- Start simple: define `icon`, `sideall` or `mesh`, `isPlaceable`, `isSolid`, `onPlaceAudio`, `onRemoveAudio`.
- If placement should require support, use `needsBase` first; move to `attachmentRequirements` for complex shapes.
- Use `onRemove` when you need weighted/multi-drop behavior; use `onRemoveType`/`onRemoveAmount` for simple cases.
- Keep `categories` consistent with existing items to get useful stockpile/UI behavior.
- Prefer `parentType` + small overrides to avoid duplicating large JSON blocks.
